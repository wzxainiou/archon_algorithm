## ✨ 基本能力

本项目使用的 YOLO (You Only Look Once) 模型具备以下核心能力：

### 1️⃣ 物体检测 (Object Detection)

- ✅ **实时检测**：能够快速识别图像中的多个物体
- ✅ **边界框定位**：为每个检测到的物体提供精确的位置信息
- ✅ **类别识别**：识别物体所属的类别（如人、车、动物等）
- ✅ **置信度评分**：提供模型对每个检测结果的确信程度

### 2️⃣ 多模型支持

本项目支持多种 YOLO 模型同时运行：

- **YOLO11 Nano (RGB)** - 最快速度，适合实时应用
- **YOLO11 Small (RGB)** - 平衡速度和精度
- **YOLOv8 Nano (Thermal)** - 热成像检测支持

## 📊 输出信息详解

每个检测结果包含以下关键信息：

### 1. 边界框 (Bounding Box)

**格式**：`[x1, y1, x2, y2]`

- **x1, y1**：边界框的左上角坐标
- **x2, y2**：边界框的右下角坐标

**示例**：
```json
{
  "bbox": [145.2, 89.7, 567.8, 423.1]
}
```

这表示检测到的物体位于：
- 左上角：(145, 90)
- 右下角：(568, 423)

**如何理解坐标**：
```
图像坐标系（原点在左上角）：
  (0,0) ─────────────────> X轴
    │
    │   (x1,y1) ┌─────┐
    │           │ 物体 │
    │           └─────┘ (x2,y2)
    │
    ↓ Y轴
```

### 2. 类别 (Class)

**类型**：字符串

识别到的物体类型，例如：
- `person` - 人
- `bus` - 公交车
- `car` - 汽车
- `dog` - 狗
- `tie` - 领带

### 3. 置信度 (Confidence)

**范围**：0.0 ~ 1.0

表示模型对该检测结果的确信程度：

- **0.9 ~ 1.0**：非常确信 ✅
- **0.7 ~ 0.9**：比较确信 ✅
- **0.5 ~ 0.7**：一般确信 ⚠️
- **< 0.5**：通常被过滤掉 ❌

**默认阈值**：本项目使用 0.25 的置信度阈值

**示例检测结果**：
```json
{
  "class": "person",
  "confidence": 0.89,
  "bbox": [12.3, 234.5, 98.7, 456.2]
}
```

解读：模型以 89% 的置信度，在位置 (12, 234) 到 (99, 456) 检测到一个人。

- JSON 格式的完整检测数据

YOLO 模型完全支持以下可视化功能（可按需添加）：

🎯 **带标注的图像输出**：
- 在原图上绘制检测框
- 标注类别名称和置信度
- 使用不同颜色区分不同类别

**示例实现**（如需添加）：
```python

from ultralytics import YOLO
```

## 💡 使用建议

### 1. 选择合适的模型

- **实时应用**：使用 YOLO11n 获得最快速度
- **精度优先**：使用 YOLO11s 获得更好的检测效果
- **资源受限**：使用 YOLO11n 或 YOLOv8n

### 2. 调整检测参数

在 `jetson_bench/cli.py` 或命令行中调整：

- `--conf 0.25`：置信度阈值（降低以检测更多对象，提高以减少误检）
- `--iou 0.45`：IOU阈值（用于非极大值抑制）
- `--imgsz 640`：输入图像大小（更大的尺寸可能提高精度但降低速度）

### 3. 理解报告中的坐标

报告中的边界框坐标是基于输入图像的像素坐标系：
- 左上角为 (0, 0)
- 向右为 X 轴正方向
- 向下为 Y 轴正方向

## ❓ 常见问题

### Q: 为什么某些物体没有被检测到？

A: 可能的原因：
1. 置信度低于阈值（默认 0.25）
2. 物体太小或被遮挡
3. 物体类别不在 COCO 80 类中
4. 光照条件差

### Q: 边界框坐标如何转换为图像位置？

A: 直接使用坐标即可：
```python
x1, y1, x2, y2 = bbox
宽度 = x2 - x1
高度 = y2 - y1
中心点 = ((x1+x2)/2, (y1+y2)/2)
```

### Q: 如何提高检测精度？

A: 建议：
1. 使用更大的模型（如 YOLO11s）
2. 提高输入图像质量
3. 调整置信度阈值
4. 使用 TensorRT 加速引擎
